# coastal-directory-shared

Shared canonical layer for all Coastal Directory sites — currently
**CoinsNearMe.co** and **ComicStoresNearMe.com**, with future sites
(SMBProFind, ChefFind, CollectibleFind, YouthSportsConnect, etc.)
to follow the same pattern.

This folder is the source of truth for:

- the page template every site uses for shop, city, state, and event pages,
- the build script (`generate.js`) that pulls data from Supabase and writes
  out HTML pages,
- the canonical CSS, JavaScript, Netlify config, and sitemap structure,
- a build verification script that catches broken pages before deploy.

## Why this exists

A 2026-05-06 architecture review of CoinsNearMe and ComicStoresNearMe
found the two sites had drifted. They started from the same playbook
but each ended up with parts the other was missing:

- ComicStoresNearMe had richer page content (FAQ schema, breadcrumbs,
  About paragraphs, neighbor-state links) but messy URL structure
  (file-and-folder duplicates) and CSS inlined into every page.
- CoinsNearMe had cleaner structure (folder URLs, external CSS) but
  bare-bones page content (no FAQ, no breadcrumbs, no cross-links).

Each was missing what the other had. This shared layer captures the
best of both, in one place, so changes are made once and inherited
by every site. Future Coastal Directory properties clone the same
canonical layer and adjust a small site config.

## How a site uses it

In Phase 2 and 3, each site's repo (`coinsnearme/`, `comicstore/`)
gets reduced to four files at its root:

1. **`site.config.js`** — the per-site config (brand, colors,
   nav links, niche-specific copy, Supabase project, GA tag).
   See `samples/coinsnearme.config.js` for a fully-worked example.
2. **`generate.js`** — a 3-line wrapper that calls the shared builder:
   ```js
   const { build } = require('../coastal-directory-shared/lib/generate');
   const config = require('./site.config');
   build(config).catch(e => { console.error(e); process.exit(1); });
   ```
3. **`netlify.toml`** — copied from `config-templates/netlify.toml`.
4. **`_redirects`** — copied from `config-templates/_redirects`,
   plus any site-specific legacy-URL redirects appended at the end.

Plus the runtime files (`style.css`, `shared.js`, etc.) are copied
into the repo as well so Netlify can serve them at the site root.

Everything else — every shop page, every state page, every sitemap —
gets generated automatically by running `node generate.js`.

## Site config — what each section means

The site config is just a plain JavaScript object with these top-level
sections. See `samples/coinsnearme.config.js` for a complete example.

| Section | Purpose |
|---|---|
| `brand` | Display name, legal name, domain, full URL. Used in titles, OG tags, schema, footer. |
| `niche` | Niche-specific labels and copy snippets. The same template renders very different content for "coin shop" vs "comic book store" because of these strings. |
| `colors` | Brand colors. Injected into pages as CSS variables (`--accent`, `--secondary`, etc.). |
| `supabase` | Supabase project URL, anon key, and table names for shops + events. **Read from environment variables, no hardcoded fallback** (security policy). |
| `analytics.gtag` | Google Analytics measurement ID. |
| `thresholds` | `minShopsForCityPage` (default 2) and `minDataFieldsForShopPage` (default 2). |
| `output.dir` | Where to write generated HTML pages. Default `./pages`. |
| `nav` | Header navigation links. |
| `footerLinks` | Footer link list (defaults to `nav` if omitted). |
| `cta` | Headline + body copy for the persistent "list your shop" banner. |

## Per-page output

For a published shop in California named "Pacific Numismatics":

- **URL:** `https://coinsnearme.co/shops/california/san-jose/pacific-numismatics/`
- **File written:** `pages/shops/california/san-jose/pacific-numismatics/index.html`
- **Schema.org JSON-LD:** `LocalBusiness` + `BreadcrumbList` + `FAQPage`
- **Meta:** title (under 60 chars), description (under 155 chars),
  OG/Twitter tags, canonical URL.
- **Body:** breadcrumb → H1 (shop name) → hero badges → service tags →
  detail rows → About paragraph → "Visit website" CTA → visible FAQ →
  "Other shops in [state]" 4-card cross-link → "Claim this shop" CTA.

## Build verification

After running `node generate.js`, run:

```sh
node ../coastal-directory-shared/scripts/verify-build.js ./pages
```

This catches:

1. Pages smaller than 1 KB (likely broken renders).
2. Orphan state folders without an `index.html`.
3. Orphan shop folders without an `index.html`.
4. Duplicate URLs (file-vs-folder collisions).
5. Sitemap entries that don't resolve to a real file on disk.

Exits with a non-zero code if anything fails, so it's safe to run
in CI as a deploy gate.
